Scala as a Object Oriented Language

In Scala we can do everything that we can do in JAVA.

Objects and Class

Class: is equivalent to Java. 

// declaring
class Example(favoriteNumber:Int, name:String="") {
    private val dadFavoriteNumber:Int = favoriteNumber * 2
    // Secondary constructor
    def this(favoritNumber:String) { 
        this(favoritNumber.toInt, "defaultName")
    }

    def dadNumber():Int = {
        return dadFavoriteNumber
    } 
}
// instantiating
val x = new Example(1, "ruiOliveira")
// using a method
x.dadNumber()

class Example(favoriteNumber:Int, name:String="")

Here we are declaring instance variables, but also the constructor. It is also possible to create as many constructors as you need, like java.

Object: is a tool that can be used in different situations. One used situation is to work has a singleton (in the nested class and object you will see that not all objects are singleton)

// a class
class Example(favoriteNumber:Int, name:String) { ... }

// a singleton object
object Example{
    var counter:Int = 10
    def counterPlusOne():Int = { counter += 1; return counter }  
    def newExample(name:String) = new Example(counterPlusOne(), name);
}
Example.newExample("Rui Oliveira")

In scala a class don't have static methods or variables, what you can do is to create a Object with the same name of the class.

Traits

Traits is equivalent to JAVA interface, but with state variables and method implementations. Note that traits are a conceptual thing resolver by the scala compiler in-lining methods. At the JVM a trait is just a Interface

Trait example

Who uses this trait must declare a value name, and will have the remaining stuff for free.

trait Person{
    val name:String;
    var nameCounter = 0
    def getName():String = {
        nameCounter += 1;
        // value name is the last thing of this block so will be returned 
        // Keep this in mind, scala don't require return keyword 
        return name;
    }

    def howManyTimesNameWasCalled() = {return nameCounter;}
}

def testTraitPerson(p:Person):Int ={return p.howManyTimesNameWasCalled();}

Object using a Trait, note that we are defining name value.

object RuiOliveira extends Person{
    val name = "rui Oliveira";
}

testTraitPerson(RuiOliveira)

Has you can see a object is a variable that uses Person so it can be used in the testTraitPerson

Class using a trait, note that we are declaring value name in the parameters now.

class OtherPerson(val name:String) extends Person

val p = new OtherPerson('pedro')
testTraitPerson(p)

Has was expected we can use traits in class's

Trait on variable. (Anonymous declaration) The class Dog don't use the trait Person, even so we can create a variable with that implementing what is needed. 

class Dog(val dogName:String)

val p = new Dog("doggy") with Person {
    val name = dogName;
}
testTraitPerson(p)

Case Class

Typically you gonna use case class for data holder class.

Case class are a class with couple of default things 

//a normal class
class Onwer( name:String)
class Phone( number:String,  owner:Onwer)
//a case class
case class Owner2( name:String)
case class Phone2( number:String, owner:Owner2)

Creating a case class you will be access:

  • default create functions: toString, copy, equal and hashcode.
  • class will be immutable by default
  • default create singleton object with method unapplay and apply functions

toString the default implementation of toString, will be very pretty for debug: "Phone("91501XXXX", Owner("Rui Oliveira"))" copy is a very handy method to create copy of an variable, modifying some attribute: 

val phoneCopy = phone.copy(number="newNumber")

apply Is a cool method that i explain in the future unaplly I don't know what exactly it does, go to google!

Case class with match/case

With a case class also you can do something like this:

val phoneCopy = phone.copy(number="newNumber")
phoneCopy match {
    case Phone("phone1", _) => ..
    case Phone("phone2", _) => ..
    case _ => ..
}
